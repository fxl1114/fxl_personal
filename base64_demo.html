Base64 图片嵌入原理演示

原理：将图片的二进制数据转换为 Base64 文本字符串，直接嵌入 HTML 中，无需外部文件。
1. 传统方式（需要外部文件）
<img src="yier.jpg" alt="一二">

缺点：需要单独发送图片文件，分享不方便

2. Base64 嵌入方式（无需外部文件）
<img src="data:image/jpeg;base64,/9j/4AAQSkZJRgABAQAA..." alt="一二">

优点：单个 HTML 文件，分享方便，图片始终可用

3. 转换过程
原始文件：yier.jpg (82,947 bytes)
↓
读取二进制数据
↓
Base64 编码
↓
添加数据 URI 前缀
↓
嵌入 HTML：<img src="data:image/jpeg;base64,...">
4. 数据 URI 格式说明
data:[MIME类型];base64,[编码数据]

示例：
data:image/jpeg;base64,/9j/4AAQSkZJRgABAQAA...
data:image/png;base64,iVBORw0KGgoAAAANSUhEUg...
data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP...
5. 优缺点对比

传统方式

✓ 文件小

✓ 浏览器缓存

✗ 需要外部文件

✗ 分享麻烦

Base64 嵌入

✓ 单文件分享

✓ 无需服务器

✗ 文件较大

✗ 无法缓存

适用场景：邮件附件、离线文档、单页应用、需要独立分享的网页
